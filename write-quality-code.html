Write Quality Code #1

Oh!! that's why they do it

Often times, when we are looking at the code of experienced developers, you notice different ways of doing something that it appears to us, why the extra effort. Then as you code you start seeing why what they did make perfect sense. Though I have no idea of many others, below are three of them I recently pick up on as a habit because I saw the importance and baggage it lifts for us:

Keeping the variables used in multiple files in one constants file

So, I was retrieving the token sent from the backend and stored in the localstorage. Token being the way of authentication, it is used in many parts of the code to check the status of authentication. But at one point, I happen to change the name of the variable used for token in the backend, and now, i have to go to all the places in the code where i have used to previous name and replace, I could also have used find and replace of vs code but that would also get the unintented instance of those variables. So, now I am convinced to put the variables name used in many places in the code in one file and export from there, and use it where necessary by importing it that way, we wuold only need to change the variable name at one place if ever we need to change it.
